# Chill Night Town 要件定義書（機能要件）v2

本章は、初期の要件定義書（完成版）をベースに、要件レビューおよび仕様検討を通じて決定した内容を反映した機能要件（1〜12）の改訂版である。現段階（卒業研究期間内）で実装する範囲を **MVP** とし、今回の実装対象外とする機能には本文中に **【将来拡張】** と明記する。

本書はシステムの利用者・開発者が仕様を理解するための文書と位置づけ、データベースのテーブル名・カラム名といった実装レベルの詳細は記載しない。データベースの詳細設計はER図・SQL定義・テーブル定義書を参照すること。

本アプリは「夜に勉強する人へ寄り添い、学習を継続しやすくする」ことを目的とした感情支援型学習アプリである。仕様・UIの判断に迷った場合は、以下のコンセプトを最優先とする。

- 夜の静かな雰囲気を感じられること
- 勉強への没入感を損なわないこと
- ユーザーに安心感や癒しを与えること
- シンプルで直感的に操作できること
- 「勉強を継続したくなる体験」を提供すること
- 勉強することを強調させないこと

## 0\. 用語定義

| 用語 | 定義 |
| :---- | :---- |
| 学習日 | 18:00〜翌5:00の1サイクルを1つの「学習日」とする。セッションの開始時刻が属するサイクルの開始日に帰属させる（例: 1月10日23:30開始・翌1:30終了のセッションは「1月10日」の記録）。カレンダー表示・目標達成判定・経験値付与はすべて学習日単位で扱う。 |
| セッション | 学習タイマーの開始から終了までの1回の学習。1セッション＝1件の学習記録として保存する。 |
| 実績学習時間 | 一時停止中および休憩フェーズ（3.1参照）を除いた、純粋な作業時間の合計。開始・終了時刻はあくまで時刻情報として別に保持し、「終了時刻−開始時刻」と実績学習時間は一致しなくてよい。街の成長・目標達成の判定にはすべて実績学習時間を使用する。 |

## 1\. 初期設定機能

### 1.1 起動

アプリを初めて起動した際、初期設定画面へ遷移する。2回目以降の起動では、初期設定が完了していればホーム画面へ直接遷移する。

未終了のセッション（3.2参照）が存在する状態で起動した場合は、保存済みの時刻情報からセッションを復元し、終了処理（成果記録）へ誘導する。

### 1.2 初期設定入力

初回起動時、以下の項目を入力・選択し、ユーザー設定として保存する。

| 入力項目 | 内容 |
| :---- | :---- |
| ニックネーム | 必須項目。以後、アプリ内の表示名として使用する。 |
| 一日の学習目標時間 | 必須項目。分単位で入力する（値域: 10分〜720分）。習慣型（6.2参照）における経験値獲得の判定基準および休憩提案（5.1参照）の基準に使用する。 |
| 好みの街の選択 | 用意された街（MVPでは2種類）から1つを選択する。以後「選択中の街」として管理する。 |
| 通知のON/OFF | 学習開始予定時間の通知設定（詳細は12章）。 |

入力完了後、以下の項目は既定値で自動的に作成され、後から設定画面（10章）で変更できる。

- 成長方式: 習慣型（6.2参照）
- 音量設定: BGM 50／環境音 50／効果音 50／鐘の音 50（9章参照）
- 感情記録: ON（3.4参照）
- タイマーモード: 黙々モード（3.1参照）

**本アプリは1端末＝1ユーザーを前提とし、ログイン機能および複数アカウント機能は実装しない。** ユーザー情報は端末内に常に1件のみ保持する。**【将来拡張】** 将来的にFirebase等のクラウドサービスと連携し、複数端末間でのデータ同期や認証機能に対応できるよう設計を考慮している。

## 2\. ホーム画面機能

### 2.1 ホーム画面表示

ホーム画面には、日付、今日（当学習日）の学習時間、目標達成状況、選択中の街のレベル、状況に応じたメッセージ（7章のNPCメッセージ）、疑似オンライン人数（11章）を表示する。画面の初期表示位置は、選択中の街の中央エリア（住宅街）とする。

画面の隅にカレンダー画面（4章）・設定画面（10章）への導線となるアイコン、おやすみボタン（13章）、および鑑賞モードボタン（2.4）を配置する。街の全景を遮る常設のナビゲーションバーは、没入感を損なうため設けない。

また、画面下部にBGMミニプレイヤー（9章）を常時表示する（タイマー計測中も表示を継続する。BGMの音量が0の場合は非表示）。

タイマー稼働中は、画面隅に計測中インジケータ（経過時間、ポモドーロモードの場合は現在フェーズ）を常時表示する。インジケータをタップするとタイマー表示（3.2）を再展開する。

### 2.2 街探索

ユーザーはスワイプ操作で背景を左右にスクロールし、選択中の街を探索できる。街は1枚の横長背景画像で構成し、シームレスに表示する。街の成長段階（6章参照）に応じて、表示される建物・オブジェクトが増加する。

### 2.3 学習開始

学習タイマーは、夜間帯（18:00〜翌5:00）のみ開始できる。時間帯の判定はホーム画面の表示時および時刻の変化時に行い、時間帯外は学習開始ボタンを非活性化したうえで、開始できない旨のメッセージを表示する。

学習開始ボタンを押すと、タイマー設定モーダル（3.1）を表示する。

---

### 2.4 鑑賞モード・横画面表示

#### 鑑賞モード（UI一括非表示）

- ホーム画面の鑑賞モードボタン（目のアイコン）を押すと、ホーム画面上のすべてのUI（日付・学習状況・レベル・NPCメッセージ・疑似オンライン人数・学習開始ボタン・計測中インジケータ・BGMミニプレイヤー・各アイコン。**鑑賞モードボタン自身を含む**）を非表示にし、夜の街の背景のみを表示する。
- 非表示中も、スワイプによる街探索（2.2）は可能とする。音声はすべて通常通り再生を続ける（音を止めて夜を閉じる「おやすみ機能」との役割の違い）。
- **画面をタップするとUIを復帰する。**
- タイマー稼働中も使用できる。表示の切り替えのみであり、計測・判定・記録には一切影響しない。
- 非表示中に休憩提案（5.1）・ポモドーロの全ループ完了・5:00自動終了（3.2）等のシステム側のイベントが発生した場合は、UIを自動的に復帰させてから該当の表示を行う。
- 鑑賞モードの状態は保存しない（アプリ再起動時は通常表示から始まる）。

#### 横画面表示（ホーム画面限定）

- 端末を横向きにすると、ホーム画面はUIを非表示にした鑑賞表示となり、**背景画像（街の全景）を1画面につなげて表示する**。全景が収まるためスクロールは不要であり、スワイプによる街探索は行わない。
- 背景の上に、以下の**最小限の情報のみ**を邪魔にならない形（半透明・小型）で重ねて表示する。
  - **日付・現在時刻**
  - **端末のバッテリー残量**
  - **タイマー稼働中のみ**：経過時間（ポモドーロモードの場合は現在フェーズも）
- **画面をタップするたびに、この情報表示の表示／非表示を切り替える**（非表示にすると街の全景のみになる）。
- 横画面は**閲覧専用**とする。情報表示の切り替え以外の操作——**タイマーの操作（一時停止・再開・終了）を含む——は一切行えない**。タイマーを操作したい場合は端末を縦向きに戻す。
- 端末を縦向きに戻すと、通常表示（UIあり）に復帰する。縦向きに戻すことが通常表示への唯一の復帰操作である。
- 横画面に対応するのは**ホーム画面のみ**とし、タイマー設定・成果記録・カレンダー・設定などの操作系画面はすべて縦固定とする。
- 横画面中にシステム側のイベント（5:00自動終了等）が発生した場合は、画面を縦向き表示へ戻したうえでUIを復帰し、該当の処理を表示する。
- 背景素材は、横向き表示時に見える上下の範囲を考慮した高さ（またはセーフマージン）で制作すること（素材制作前の必須確認事項）。

**【将来拡張】** ホーム画面以外の画面の横画面対応。

### 2.5 今夜の天気の選択（ホーム画面）

夜の天気（3.4の11種類）は、**学習を開始しなくてもホーム画面から選択できる**。選択した時点から、その学習日の背景演出（8章）・環境音（9章）へ即座に反映する。

- **その学習日の天気が未選択の場合、18:00以降のホーム画面に「今夜はどんな夜？」等の控えめな案内カードを表示する。** カードには**対象の学習日**（例: 1/10（金）の夜）を併記し、どの日の記録として扱われるかを明示する。
- 案内カードは**モーダルではなく、操作を妨げない形**で表示する。閉じることができ、閉じても天気は後から選べる。**天気の選択は必須ではない**（ただし学習タイマーの開始時には必須。3.1）。
- **ホーム画面の情報表示（日付・今夜の学習仲間・今夜の学習時間と同じ並び）に、その夜の天気を1行で常時表示する**（例: 🌧 雨音の夜／未選択時は「今夜の天気を選ぶ」）。この行をタップすると天気を選択・変更できる。**専用の常設ボタンは設けない**（ホーム画面は既に多くのUIを抱えており、街の全景を遮らない方針〈2.1〉に反するため）。天気は「今夜の状態」を表す情報であり、日付・学習時間と同じ並びに置くのが自然である。
- 18:00より前にホーム画面を開いていた場合は、**18:00に到達した時点でカードを静かに表示する**。ただし鑑賞モード中（2.4）・横画面中・タイマー設定モーダル表示中は表示せず、通常表示へ戻った時点で表示する。
- **選択済みの学習日にはカードを表示しない**（情報表示の天気の行からの変更は常に可能）。
- 天気の選択・変更は、この画面のほかタイマー設定（3.1）・学習成果記録（3.4）からも行える。**1晩＝1天気**とし、いずれの経路から選び直しても、**その学習日の天気を上書きする（最後の選択が残る）**。
- 天気は**学習日単位**で保持し、前日以前の天気は引き継がない（8章）。変更できるのは**その学習日が終わる（5:00）まで**であり、学習日が変われば新しい夜となる。過去の夜の天気は変更しない（カレンダーからも夜の天気は編集対象外。編集できるのはタグ・メモのみ・4.1）。

> **補足**: 本機能は当初【将来拡張】としていたが、天気がタイマー開始まで選べないと、街に帰ってきただけでは夜の空気が立ち上がらず、コンセプト（またこの世界に帰ってきたい夜の街）と噛み合わないためMVPへ移した。ただし起動時に選択を強制すると入口に関門を作ることになるため、**非強制・非モーダル**とする。

## 3\. 学習タイマー機能

学習タイマーは独立した画面ではなく、ホーム画面（夜の街）の上に重なる表示として構成する。学習中もユーザーは夜の街の中に留まり続ける。

### 3.1 タイマー設定（開始前）

学習開始ボタン押下後、タイマー設定モーダルを表示する。設定項目は以下の通り。

| 設定項目 | 内容 |
| :---- | :---- |
| 今夜の天気 | 11種類（3.4参照）から1つを選択する。**その学習日の天気がまだ選ばれていない場合は必須**。すでに選択済みの場合（ホーム画面 2.5 での選択、または同じ学習日の過去の学習開始時の選択）は、その選択を引き継いで表示し、変更も可能。選択した天気は、**開始ボタンを押した時点で**その学習日の天気として確定し（1晩＝1天気・上書き）、ホーム画面・学習中の背景演出（8章）および環境音（9章）に反映される。 |
| タイマーモード | 「黙々モード」「ポモドーロモード」から選択する。前回選択したモードを記憶し、次回は選択済みの状態で表示する。 |
| （黙々モード）予定学習時間 | 予定する学習時間を分単位で設定する（値域: 10〜660分、既定値60分）。 |
| （ポモドーロモード）作業時間・休憩時間・繰り返し回数 | 作業時間（値域: 5〜120分、既定値25分）、休憩時間（値域: 1〜30分、既定値5分）、繰り返し回数（値域: 1〜10回、既定値1回）を設定する。 |

**設定値はモードとあわせて記憶し、次回は前回の値を入れた状態で表示する。** 毎回同じ時間を設定し直す手間を省くため。既定値は初回のみ使用する。

記憶は**入力を終えた時点**で行い、開始しなくても次回に引き継ぐ。設定を見直しただけで閉じた場合にも、その値を覚えていてほしいため。ただし**値域外の値は記憶しない**（次回に開始できない値が入った状態で開くのを避けるため）。

#### タイマーモード

- **黙々モード**: 設定した予定学習時間に向かって連続で計測する。従来型のタイマー。
- **ポモドーロモード**: 作業フェーズと休憩フェーズを交互に繰り返す。作業フェーズを設定した繰り返し回数だけ行い、**休憩フェーズは作業と作業の間にのみ挟む（最後の作業フェーズの後に休憩は行わない）**。すなわち繰り返し回数 n の構成は「作業 →（休憩 → 作業）× (n−1)」となり、総経過時間は 作業×n ＋ 休憩×(n−1) となる（繰り返し回数が1回の場合、休憩は発生しない）。フェーズの切り替わりは自動で行い、切り替わり時には控えめな効果音（鐘とは別の音）を再生する。**最後の作業フェーズの完了をもって全ループ完了**とし、終了演出（3.3）を行う。**【将来拡張】** アプリがバックグラウンドにある間のフェーズ切り替わり通知（ローカル通知の予約）は今回の実装対象外とし、切り替わり音の再生はアプリがフォアグラウンドにある間のみ行う。

### 3.2 タイマー計測

学習タイマーは開始・一時停止・再開が可能で、1回のセッション中に繰り返し操作できる。タイマー開始時に設定した**予定学習時間**（ポモドーロモードの場合は作業時間×繰り返し回数）と、実際に学習した**実績学習時間**をそれぞれ記録する。これにより「予定60分に対して実績40分」のような差分を保持できる。

#### 計測方式

経過時間は、開始・一時停止・再開の各時刻を操作のたびに端末へ保存し、**現在時刻との差分**から算出する（カウンター方式は用いない）。バックグラウンドでの処理実行は行わないが、この方式によりアプリを離れて戻った場合でも経過時間は正しく引き継がれる。

#### 中断からの復元

アプリの強制終了・クラッシュ・端末再起動などでセッションが終了処理を経ずに中断された場合、次回起動時に保存済みの時刻情報からセッションを復元し、「前回のセッションが終了していません」として終了処理（成果記録）へ誘導する。復元時点で5:00を過ぎていた場合は、5:00終了のセッションとして扱う。

#### 5:00到達時の自動終了

タイマー稼働中（一時停止中を含む）に翌5:00に到達した場合、自動的に終了演出（3.3）へ移行し、実績学習時間は5:00までとする。この仕様はタイマー開始時に画面上で明示する。

#### 実績1分未満のセッションの破棄

実績学習時間が1分未満の状態で終了（手動終了・5:00自動終了・中断復元時を含む）した場合、終了演出・成果記録へは進まず、**セッションを保存せずに破棄**する。「1分未満のため記録されませんでした」等の控えめなメッセージを表示してホーム画面へ戻る。街への加算・成長判定は行わない。誤って開始してしまった場合は、すぐに終了すれば記録が残らない。

なお、**その学習日の天気（2.5）は破棄しない**。天気の選択は学習セッションから独立した行為であり（ホーム画面からも選択できる）、セッションの破棄によって選び直しを強いるのは不自然なため。天気を変えたい場合はホーム画面の天気の行から変更できる。

#### 端末時刻の変更への耐性

経過時間の差分計算の結果が負の値になった場合（ユーザーによる端末時計の変更等）は、0に丸めて扱う。

#### 画面スリープの防止

タイマー稼働中（一時停止中を含む）は、OSの画面スリープを防止する。あわせて、横画面表示中および鑑賞モード中（2.4）もスリープを防止する。それ以外の場面ではOS標準のスリープ動作に従う。

#### タイマー表示の折りたたみ

タイマー表示は「×」操作で折りたたみ、街（ホーム画面）へ戻ることができる。計測は継続され、計測中であることはホーム画面のインジケータ（2.1参照）で表示する。学習を「中止」して記録を残さない機能は用意しない。終了は必ず終了演出・成果記録を経由する（感情・タグ・メモは任意項目のため、すぐに保存して閉じることができる）。

#### タイマー稼働中の他画面利用

タイマー稼働中も、他の画面（カレンダー・設定等）へ移動できる。ただし、**進行中のセッションの判定・記録に影響する操作、および破壊的操作は、タイマー稼働中は変更不可とする**。詳細は10章を参照。

### 3.3 終了演出

学習終了時には鐘の音による終了演出を行う。鐘の音の再生中は、BGM・環境音の音量を一時的に下げる（ダッキング）。

### 3.4 学習成果記録

学習終了後、以下の内容を入力・確認し、1件の学習記録として保存する。

| 記録項目 | 内容 |
| :---- | :---- |
| 夜の天気 | その夜に選択済みの天気を表示する。ここで変更することもできる（「振り返ってみたら今夜は嵐の夜だった」を許容する）。**1晩＝1天気**のため、ここでの変更はその学習日の天気そのものを変更する（同じ夜の他の学習記録の表示にも反映される）。 |
| 感情 | 下記11種類から1つを選択する。感情記録の設定がONの場合のみ記録する。 |
| 学習内容タグ | 複数選択できる。標準タグに加え、自由入力によるオリジナルタグ（マイタグ）にも対応する（詳細は本章末尾）。 |
| 振り返りメモ | 自由入力、任意項目。 |
| 開始・終了時刻 | 学習を開始・終了した日時を記録する。 |
| 予定・実績学習時間 | 3.2で記録した内容を保存する。 |

ポモドーロモードの場合も、1式（全ループ）で1件の学習記録とする。ループごとに記録を分割しない。

#### 成果記録画面から離脱した場合

成果記録を保存せずに画面から離脱した場合、セッション（開始・終了時刻、予定・実績学習時間）は自動的に保存・確定する。感情・タグ・メモは未入力（空）のまま確定する。学習した時間（街の成長への加算分）が失われることはない。その夜の天気は開始時に確定済み（2.5 / 3.1）のため、離脱による影響を受けない。

#### 夜の天気（全11種類）

✨ 星空の夜／🌙 月灯りの夜／🌕 満月の夜／🌑 闇夜／☁️ 雲間の夜／🌧️ 雨音の夜／⛈️ 嵐の夜／❄️ 雪明かりの夜／🌃 静寂の夜／🌫 霧の夜／🎆 花火の夜

夜の天気は気象データではなく、「その夜がどんな夜だったか」を表す心象風景として選択するものである。実際の空を見て選んでも、気分で選んでもよい。**【将来拡張】** 実際の気象データ（天気API）との連携による背景演出の補助。

#### 感情（全11種類、3カテゴリ）

- **ポジティブ**: 😊 達成感／🔥 集中できた／💪 頑張れた／😄 楽しかった
- **ニュートラル**: 😌 穏やかだった／😶 いつも通り／😴 眠かった
- **ネガティブ**: 😔 疲れた／😞 少し落ち込んだ／😥 不安だった／😣 思うように進まなかった

#### 学習内容タグ（標準タグ＋マイタグ）

標準タグは以下の5種類とする（マスタデータ）。

**資格勉強／レポート・課題／暗記・復習／プログラミング／読書**

「その他」は標準タグに含めない。**タグは任意項目であり何も選ばずに保存できる**ため、「その他」を選ぶことと何も選ばないことの情報量が変わらず、記録を振り返る際に得るものがないため。分類しきれない内容はマイタグとして具体的な名前で登録できる。

これに加え、ユーザーは自由入力でオリジナルの「マイタグ」を登録できる。マイタグの仕様は以下の通り。

- 初めて入力したタグは以後「マイタグ」として保存し、次回以降は標準タグと同様に選択できる。
- 同一名称のマイタグは重複登録しない。**標準タグと同名のマイタグも登録できず**、その場合は「すでに同じ名前のタグがあります」等のエラーメッセージを表示する（登録時・編集時で挙動を統一する）。
- マイタグは設定画面（10章）から編集・削除できる。**名称の変更は、過去の学習記録の表示にも反映される。**
- 削除は**論理削除**とし、データそのものは残す。削除済みのマイタグは新規の選択肢には表示されないが、過去の学習記録ではそのまま表示され続ける。
- **削除済みのマイタグと同名のタグが新たに入力された場合は、削除済みのタグを復活させる**（削除フラグを解除して選択肢に戻す）。新規タグとして別に作成しない。
- マイタグは1ユーザーあたり最大20件まで登録できる。**削除済み（論理削除状態）のマイタグは、この上限のカウントに含めない。**
- タグ名の上限は20文字、振り返りメモの上限は500文字とする（いずれもアプリ側の入力検証で担保する）。

## 4\. 学習記録カレンダー機能

### 4.1 日別記録の閲覧・編集

カレンダー上で日付を選択すると、その学習日の学習時間、感情、夜の天気、タグ、メモを閲覧できる。日付への帰属は「学習日」の定義（0章）に従う。記録が存在しない日は、その旨が分かるデフォルト表示を行う。

日別詳細からは、各セッションの**学習内容タグと振り返りメモを後から編集できる**（セッションを長押しして編集する）。過ぎた夜の記録を整えて振り返りの質を高められるようにするためである。一方、**感情・学習時間・夜の天気は編集対象としない**。感情はその夜のその時の気持ちのスナップショットであり、後から書き換えると記録が事実でなくなるため閲覧のみとする。学習時間は実績の改ざんになるため変更できない。タグ・メモの入力仕様（標準タグ／マイタグ・重複・上限・メモ500文字）は学習成果記録（3.4）に準ずる。

### 4.2 月次サマリー

月単位で以下の集計情報を表示する。

- 総学習時間
- 学習回数
- 最も多かった感情
- 最も多かった夜の天気
- 感情別の記録回数

なお、**夜の天気アルバム**は専用のデータを持たず、これまでに選択された夜の天気（2.5）を都度集計して表示する方針とする。アルバムの充実度に応じた演出（6章参照）も、この集計結果に基づいて判定する。

- 集計の対象は**学習記録が存在する学習日のみ**とする。天気だけ選んで学習しなかった夜はアルバムに含めない。
- **1晩＝1天気**のため、「最も多かった夜の天気」は**夜の数**で数える（1晩に複数回学習しても1回と数える）。
- 感情は学習記録ごとに保存されるため、感情に関する集計は学習記録単位で行う。感情が未入力の記録は感情の集計から除外する（学習時間・学習回数の集計には含める）。

## 5\. 学習セッション管理機能

### 5.1 学習時間超過時の休憩提案

その学習日の実績学習時間の合計が**最初の休憩提案の基準**に到達した時点で、休憩を提案する画面（タイマー表示上の重ね表示）を表示する。その後も学習を続けている場合、**超過60分ごと**に再表示する。この機能はON/OFFを設定できる（頑張りすぎ防止設定）。

**最初の休憩提案の基準**は、「一日の学習目標時間」と「セッション開始時点の実績合計 ＋ 今回のセッションで設定した予定学習時間」のうち**大きい方**とする。ユーザーが自分で決めた学習時間の途中では提案を出さないためであり（例: 目標60分でも今回90分と設定したなら、90分ぶんを終えるまで提案しない）、目標が予定より短い場合に「決めた分」の途中で割り込んで急かすことを避ける。予定学習時間は黙々モード＝設定した分数、ポモドーロモード＝作業時間×繰り返し回数とし、**2つのモードで仕様を統一**する。

表示と同時に、休憩の時間だと気づけるよう**それとなく柔らかい効果音**（9章の「効果音」分類。急かすアラームのようには鳴らさない）を1回再生する。**鐘の音は使用しない。** 鐘は学習終了の演出（3.3）に用いる音であり、休憩提案で鳴らすと学習を継続した場合に1晩で2回鳴って意味が混ざり、鐘が「終わりの合図」として機能しなくなるため。ポモドーロのフェーズ切り替わり音（3.1）と同じ考え方に従う。

選択肢は2段階で提示する。目標に届いた瞬間に選択肢を一度に並べると、静かな体験の中で判断の負荷が高くなるため、まず大きな問いを1つ示し、続きを選んだ場合にのみ細かい選択を示す。

**1段目**

| 選択肢 | 内容 |
| :---- | :---- |
| 今夜はここまでにする | 学習を終了する。終了演出（3.3）を経て学習成果記録（3.4）へ進む。タイマー表示の終了操作と同じ動作 |
| まだ続ける | 2段目を表示する |

**2段目**

| 選択肢 | 内容 |
| :---- | :---- |
| 休憩する | タイマーを一時停止する。再開はユーザーの操作による |
| このまま続ける | 表示を閉じ計測を継続する。以後、超過60分ごとに再表示する |
| 時間を決めて続ける | 延長を宣言する（5.2） |

**「今夜はここまでにする」を用意するのは、目標に届いた瞬間は学習を終える意思が生じやすい場面であるため。** これが無いと、終了はタイマー表示の終了操作からしか行えず、提案を閉じ、タイマーを開き直す必要がある。

**続けることを選んだ場合、次に提案を表示する基準は必ず先送りする。** 「休憩する」を選んだ場合も同様に**超過60分後**を次の基準とし、再開した直後に再表示されることはない。休憩は実績学習時間に加算されないため、基準を先送りしなければ提案を閉じた瞬間に表示条件を満たしたままとなり、再び表示されてしまう。

| 選択 | 次に表示する基準 |
| :---- | :---- |
| 休憩する | 超過60分後 |
| このまま続ける | 超過60分後 |
| 時間を決めて続ける | 宣言した時間の経過後（5.2） |

**ポモドーロモード中の作業フェーズには割り込まない。** ポモドーロモードでは、休憩提案の表示は休憩フェーズまたはループ完了のタイミングで行う（休憩が組み込まれたモードであるため、作業中への割り込みは行わない）。

### 5.2 延長宣言（追加学習時間設定）

目標時間を超過してさらに学習を続けたい場合、分単位で学習の延長を宣言できる（値域: 5〜120分）。宣言した時間内は休憩提案（5.1）を表示しない。宣言時間を使い切ってなお学習を続けている場合は、再び休憩提案を表示する。

延長宣言は**休憩提案の表示制御にのみ使用し、目標達成・経験値の判定（6.2）には影響しない**。宣言した時間は当日（当学習日）のみ有効とする。

**【将来拡張】** 当初検討していた「集中モード機能（学習中の通知停止、およびアプリ利用制限機能）」は、MVPで停止できる通知が自アプリの学習開始リマインダーのみであり実質的な効果が薄いこと、アプリ利用制限にはOS側の機能との連携が必要であることから、機能ごと今回の実装対象外とする。代替として、より深い集中を求めるユーザー向けに、OSのおやすみモード・集中モードの利用を案内する文言を画面内に配置する。

## 6\. 街成長機能

### 6.1 学習時間に応じた街の成長

学習を継続することで、選択中の街が成長する。街の成長段階は**現段階（MVP）では全5段階（Lv.1〜Lv.5）とし、街ごとに個別に管理する。【将来拡張】** 将来的には10段階以上へ拡張可能な設計とする。

街の成長に伴い、建物・背景・オブジェクト・NPC・景観演出が変化するほか、4章で述べた夜の天気アルバムの充実度に応じた装飾演出も追加される。

街の成長状態（レベル・累計学習時間・経験値）は、**街ごとに個別管理**する。街を切り替えても各街の育成状態は保持され、再度その街を選択した際は保存済みの状態から再開する（6.4参照）。

MVPで用意する街は**2種類**とする（テーマ・デザインは素材制作時に決定する）。

各街には、任意で**サブタイトル**（上限20文字）を設定できる。サブタイトルは街名と組み合わせて「夜の街（期末テスト）」のように表示する。未入力の場合は街名のみ表示する。サブタイトルは街の育成情報に紐づく汎用項目であり、成長方式に関わらずいつでも設定・変更できる。

#### 成長処理の共通ルール

- 成長判定は、学習記録の保存（3.4）を契機に実行する。
- **累計学習時間は、選択中の成長方式に関わらず、常に選択中の街へ加算する。** 経験値は習慣型を選択中のみ付与する。
- レベルアップ判定は、条件を満たす限り繰り返し行う（一度の保存で複数レベル分の条件を満たした場合は、該当レベルまで上昇させる）。
- **街のレベルは一度上がったら下がらない**（過去に到達した最高レベルを維持する）。成長方式の切り替えや目標学習時間の変更によって判定基準が変わった場合も、このルールを適用する。
- Lv.5（街完成）到達時には、一度だけ完成演出を表示する。完成後も実績（累計学習時間・経験値）の加算は継続するが、レベルはLv.5で打ち止めとする。

### 6.2 街の成長方式設定

街の成長方式は、以下の2種類から選択する。成長方式は**アプリ全体で共通の設定**とし、街ごとに個別設定することはできない（成長の実績値は6.1の通り街ごとに個別管理する）。既定値は**習慣型**とする。

#### ① 習慣型 —— 毎日コツコツ続けて街を育てる

ユーザーが設定した「一日の学習目標時間」（1.2参照）を達成した学習日に、街の経験値を1獲得する。目標未達成の日は経験値を獲得しない。

- 目標達成の判定は**学習日単位の実績学習時間の合計**で行い、学習記録の保存のたびにチェックする（1日に複数セッション行った場合は合算する）。
- 経験値の付与は**1学習日につき最大1回**とする。達成した時点（達成させたセッションの保存時）に付与する。
- **一度付与した経験値は、その後同じ学習日内に目標時間が変更されても取り消さない。**

例）1日の目標時間が120分の場合: その学習日の実績合計が120分に達した時点 → 経験値+1（以降その学習日は加算なし）

経験値が一定数貯まると街のレベルが上昇する。レベルアップに必要な経験値は**一律5**とし、全ての街で共通の基準値を用いる。基準値はマスタデータとして保持し、実装後のバランス調整で変更できるようにする。

#### ② プロジェクト型 —— 目標に向かって街を完成させる

「期末テストまでに10時間」「資格試験までに30時間」のように、現実の目標に紐づいた**目標学習時間を街ごとに設定**する。学習した時間はそのまま選択中の街の累計学習時間へ加算され、目標学習時間の達成＝街の完成（Lv.5）となる。

街のレベルアップに必要な累計学習時間は、設定した目標学習時間を現在実装している5段階で均等に割った値を基準として自動計算する。

例）目標学習時間を50時間に設定した場合 Lv.1（初期状態）→ Lv.2：10時間 → Lv.3：20時間 → Lv.4：30時間 → Lv.5（街完成）：50時間

- 目標学習時間は、プロジェクト型を選択する際、および街選択画面（6.4）から設定・変更できる。
- 目標学習時間を途中で変更した場合、レベルアップ基準は再計算されるが、6.1の共通ルールによりレベルは下がらない（基準の引き下げにより新たに条件を満たしたレベルへは即時上昇する）。
- **【将来拡張】** 目標に期限（日付）を設定する機能。期限管理はユーザーを急かす方向の機能になり得るため、MVPでは合計時間の進捗のみを静かに表示する。

いずれの方式を選択した場合も、成長方式の切り替えはいつでも可能であり、切り替え前に蓄積した累計学習時間・経験値は削除せず両方とも保持し続ける。街の成長判定には、その時点で選択している成長方式の実績値のみを使用する（表示レベルは6.1のルールにより過去最高値を維持する）。

### 6.4 街の切り替え

ユーザーは複数の街を切り替えて育成できる。各街の育成進捗は個別に保持され、切り替え後も失われない。学習中（タイマー稼働中）は街の切り替えを行うことができない。街選択画面では、各街のサブタイトルの設定・変更、およびプロジェクト型の目標学習時間の設定・変更も行える。

## 7\. NPC機能

### 7.1 状況に応じたNPCメッセージ表示

NPCは、学習開始・学習終了・目標達成といった基本的なタイミングで固定メッセージを表示し、ユーザーの学習を励ます。

- **現段階（MVP）ではNPCは1体のみ実装する。**
- メッセージは固定文方式とし、発生したタイミング（学習開始／学習終了／目標達成）に応じて用意した候補群の中から、条件に一致するものをランダムに1件選択して表示する、単純な条件マッチ方式とする。ログイン日数・夜の種類等を組み合わせた複雑な条件判定は行わない。
- **学習終了・目標達成のメッセージは、学習成果記録（3.4）で選択された感情に応じて出し分ける。** 感情ごとに候補群を用意し、その中からランダムに1件選択する。条件は「タイミング＋感情」の単純な一致のみとし、他の条件と組み合わせることはしない。感情が未選択の場合、および感情記録の設定（10.7）がOFFの場合は、感情を問わない候補群から選択する。
- 「目標達成」のタイミングは、学習記録の保存時に目標達成判定（6.2①）が成立した時点とする。
- 用意するメッセージ量は、1体あたり20〜30種類程度を基準とし、感情ごとの出し分け分を加えて50種類程度とする。感情ごとの候補は当初1件ずつとし、運用しながら追加できるものとする（データを追加するだけで増やせる構造とする）。

例） 学習開始：「急ぐ必要はありません。夜は長いのですから。」 学習終了（感情=疲れた）：「お疲れさまでした。今夜はもう、何もしなくていい夜です。」 学習終了（感情=いつも通り）：「いつも通り。それを続けられることが、いちばん難しいのですよ。」 目標達成（感情=思うように進まなかった）：「手応えがなくとも、時間は確かに積み上がりました。届いていますよ。」

感情に応じた出し分けを行うのは、**目標に届いた夜でも手応えが無いことがあり、その食い違いを受け止める言葉こそが必要**なため。数字上の達成だけを祝う言葉は、そうした夜のユーザーを置き去りにする。

**【将来拡張】** 将来的には、街ごとのNPC追加や、固定メッセージ方式に代わるAI住人機能への拡張を予定している。NPCおよびメッセージはあらかじめ拡張しやすいデータ構造で管理しており、表示タイミングの種類も自由に追加できる。

## 8\. 深夜演出機能

時間帯に応じて空の色やエフェクトなど画面の雰囲気を変化させ、夜の街としての世界観を演出する。

背景演出に使用する「夜の天気」は、**その学習日に選択された天気**とする。選択の経路はホーム画面（2.5）・タイマー設定（3.1）・学習成果記録（3.4）の3つで、いずれの場合も選択された時点から、ホーム画面・学習中の背景演出および環境音（9章）へ即座に反映する。その学習日にまだ天気が選択されていない場合は、天気演出のないニュートラルな夜空（時刻連動の空の色のみ）を表示する。前日以前の天気は引き継がない。

天気は**学習日ごとに1つだけ保持する**（`daily_night_weather`）。**1晩＝1天気**であり、その学習日のあいだ何度でも選び直せて、**最後に選択された天気がその夜の天気として残る**。学習セッションは天気を個別に持たない（同じ夜の学習はすべて同じ天気となる）。

## 9\. BGM・効果音機能

雨音・波音・森の音・夜風・Lo-Fi BGMなど、複数の音源素材を用意し、時間帯や選択中の夜の天気（8章と同じ参照元）に応じて自動再生する。

アプリ内の音は以下の4種類に分類し、それぞれ**0〜100の音量**をユーザーが設定できる（既定値はすべて50）。

| 分類 | 内容 |
| :---- | :---- |
| BGM | Lo-Fi等の音楽。 |
| 環境音 | 雨音・波音・夜風など、時間帯・夜の天気に応じて流れる背景音。 |
| 効果音 | ボタン操作音・画面遷移音・ポモドーロのフェーズ切り替わり音などのUI音。 |
| 鐘の音 | 学習終了演出の音（3.3参照）。 |

- 音量を変更した際は、現在の音量が分かるようにプレビュー音を再生する。ただしプレビュー音を鳴らすのは**効果音・鐘の音のみ**とする。BGM・環境音は連続再生される音で、音量変更が再生中に即反映されて耳で分かるため、別途のプレビュー音は鳴らさない。
- **音量0の音は、再生処理自体を行わない**（ミュートと同義）。
- 鐘の音の再生中は、BGM・環境音の音量を一時的に下げる（ダッキング）。

#### BGMの再生とミニプレイヤー

- **BGMはホーム画面の初回表示時点では停止（デフォルト停止）とし、自動再生はしない。** 起動と同時に音が鳴るのを避け、聴くかどうかをユーザーに委ねる方針。ミニプレイヤーには最初の曲（シャッフル済みプールの先頭）の曲名を表示しておき、ユーザーが再生ボタンを押すと**フェードインで静かに再生を開始**する（急に鳴らさない）。
- 再生順はBGMプール（マスタデータのうちBGMに分類された音源）の**シャッフル再生**とし、曲が終わると自動的に次の曲へ進む。
- ホーム画面下部に小型の**ミニプレイヤー**を常時表示し（タイマー計測中も表示）、以下の操作・表示を提供する。
  - 再生中の**曲名・アーティスト名（クレジット表記）**の表示。長い曲名はスクロール表示とする。フリー音源のクレジット表記義務はこの欄で満たす
  - **再生位置の進捗バー**（曲がどれくらい進んだかを視覚的に示す）
  - **一時停止／再開**（対象はBGMのみ。環境音は天気演出の一部として流れ続ける）
  - **スキップ**（次の曲へ進む。停止中は曲を切り替えるのみで再生はしない）
  - **巻き戻し**（再生中の曲の頭に戻る。前の曲へは戻らない）
- BGMの音量が0の場合、ミニプレイヤーは非表示とし、BGMの再生処理も行わない。
- プレイリストの作成・曲の指定選択は行わない。
- **【将来拡張】** バックグラウンド再生（アプリを閉じた状態での再生継続）は行わない。

環境音は従来通り、時間帯・夜の天気に応じた自動選択・自動再生のみとし、操作UIは設けない。

**現段階ではユーザーによる個別の音源選択は行わない。【将来拡張】** 将来的にユーザーが好みの音源を個別に選択できる機能へ拡張する予定であり、拡張しやすいデータ構造をあらかじめ用意している。

### 10.10 アプリ内データ削除（初期化）

本機能は、アプリ内データの完全初期化機能である。実行すると、端末内に保存されている全てのユーザーデータを削除し、アプリを初回起動時と同じ状態に戻す。削除対象は以下を含む。

- ユーザー設定（ニックネーム、初期設定、各種設定）
- 学習記録・感情記録・タグ付け情報
- 街の育成状況（サブタイトルを含む）
- 夜の天気アルバム情報（学習記録から動的に生成される情報を含む）
- マイタグ（ユーザーが登録したもの）
- その他アプリ内で保存している全てのデータ

削除実行時には確認ダイアログを表示し、ユーザーが最終確認を行った場合のみ初期化を実行する。**削除されたデータは復元できない。** 初期化の完了後は、その場で初期設定画面へ遷移する。

なお、街・夜の天気・感情・NPC・NPCメッセージ・音源・成長段階の基準値などのマスタデータは、ユーザーが作成したものではないため削除対象に含まれない。

## 11\. 疑似マルチプレイ機能

ホーム画面等に、他ユーザーが同時に学習しているかのような演出として、疑似的なオンライン人数を表示する。

- **現段階（MVP）では、表示する人数はランダム生成による演出とし、実際のオンラインユーザー数は取得しない。**
- アプリ起動時に一度だけ、**3〜27人**の範囲でランダムな人数を生成し、そのアプリ起動中は同じ人数を表示し続ける。
- 人数のデータは保存しない（永続化不要な演出用データとして、アプリの動作中のみ保持する）。
- 表示文言は静かな世界観に合わせる（例:「今夜は12人が街を灯しています」）。

**【将来拡張】** 将来的にはFirebase等のクラウドサービスを利用し、実際のオンラインユーザー数を表示する機能へ拡張する予定である。

## 12\. システム通知機能

学習開始予定時間になったら、OSのローカル通知機能を用いて通知を送信する。通知のON/OFFおよび通知時刻は設定情報として保存する（10.3参照）。

- 通知は**毎日同時刻の繰り返し**とする。通知時刻は **17:30〜翌4:30** の範囲で設定できる。この範囲は、学習タイマーを起動できる夜間帯（18:00〜翌5:00）を基準に、開始前の準備を促せるよう前側を17:30まで広げ、5:00到達で自動終了する仕様に配慮して後側を4:30までとしたものである。範囲外の時刻は設定できない（開始できない時間に開始を促す不整合を避けるため）。
- **18:00より前（17:30〜17:59）に設定された通知**は、まだ学習を開始できないため、通常の開始リマインドではなく「夜が目覚めるまであと◯分」のような、夜の開始までのカウントダウンを伝えるメッセージとする。◯分は「18:00−通知時刻」で一意に定まるため、スケジュール登録時に静的に確定できる。**18:00以降（〜翌4:30）**に設定された通知は、通常の学習開始リマインドメッセージとする。
- 通知の実現方式は、設定保存時にOSへローカル通知をスケジュール登録し、設定OFF時・時刻変更時には登録済みの通知を解除・再登録する方式とする（アプリが時刻を監視するのではなく、発火はOSが行う）。
- OSの通知許可が未取得の場合、設定をONにした時点で許可を要求する。**許可が拒否された場合は、アプリ内の通知設定をOFFに戻し、OSの設定画面から変更できる旨を表示する。**

**現段階（MVP）では、通知の送信履歴は保存しない。** 送信済みフラグや送信ログの管理は行わず、通知設定（ON/OFF・時刻）のみを保持する。**【将来拡張】** 将来的に通知の利用状況分析や送信履歴の表示機能を実装する場合は、送信履歴を保存する仕組みを別途追加する方針とする。

## 13\. おやすみ機能

一日の学習の終わりに「夜を閉じる」ための機能。ホーム画面のおやすみボタン（月アイコン等。カレンダー・設定アイコンと並べて配置）から実行する。

- ボタンを押すと、「眠りにつきますか？」等の世界観に合わせた文言で確認表示を行う。
- 確定すると、BGM・環境音をフェードアウトし、画面を暗転させる（**シンプルな暗転のみ**とし、街の灯りが消えていく差分演出は行わない）。暗転画面にはNPCのおやすみメッセージを一言表示する。
- 暗転中はすべての音の再生処理を停止する。ユーザーはこの状態で安心してアプリを閉じられる。
- 暗転画面をタップするとホーム画面へ復帰する。このとき**環境音はフェードインで再開する**が、**BGMは停止のままとする**（BGMはデフォルト停止であり、再生はユーザーが再生ボタンで始める。9章）。一度おやすみした後でも、いつでも戻れる。
- **タイマー稼働中（一時停止中を含む）はおやすみボタンをグレーアウト**し、タップ時に「学習中はおやすみできません。先に学習を終えてください」等の控えめなメッセージを表示する。

なお、iOSではアプリが自分自身を終了させること（プログラムによる自己終了）がAppleの審査ガイドラインで認められていないため、本機能はOSレベルのアプリ終了は行わない。「音を止め、画面を閉じ、安心してアプリを離れられる状態を作る」ことを目的とした演出機能と位置づける。本機能は演出のみであり、保存するデータを持たない。
