# 開発用テストボタン一覧

開発中の動作確認のために設置している UI と、暫定（仮実装）のまま残っている操作をまとめる。
**リリース前に「撤去チェックリスト」を必ず消化すること。**

`__DEV__` 限定のものは Expo の開発ビルドでのみ表示され、本番ビルドには出力されない。

---

開発用のボタンはホーム画面の下部に「開発用パネル」（`src/app/index.tsx` の `DevPanel`）としてまとめている。

---

## 1. データ初期化（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「開発用: データ初期化して初期設定へ」 |
| 挙動 | 全ユーザーデータを削除 → 設定を再読込 → 初期設定画面（`/setup`）へ遷移 |
| 実装 | `src/app/index.tsx` の `DevPanel` / `src/db/repositories/maintenanceRepo.ts` の `resetUserData()` |

**補足**

- 削除は `DELETE FROM user` のみ。スキーマの `ON DELETE CASCADE` により、ユーザーに紐づく全データ（学習記録・進捗・各種設定・マイタグ・計測状態等）が連動削除される。マスタデータは残る
- 初期設定フローを何度も試すために用意したもの
- 正式なデータ初期化（要件10.10・確認ダイアログ・タイマー稼働中の禁止を含む）は **Phase 6** で設定画面に実装する。その際に本ボタンは撤去してよい

## 2. レベルの循環（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「レベル: Lv{n}」 |
| 挙動 | 押すたびに選択中の街のレベルを **1→2→3→4→5→1** と循環させる。背景アート・Lv表示に即反映。**レベルに応じて選択中の成長方式の実績値も辻褄を合わせる** |
| 実装 | `src/app/index.tsx` の `DevPanel` / `src/db/repositories/devRepo.ts` の `cycleTownLevel()` |

**補足**

- 目的: **各レベルの見た目（灯り・背景アート）の確認**と、**レベルアップ・完成演出のやり直し**を1つのボタンで賄う（従来は「Lv切り替え（表示プレビュー）」と「街の育成をやり直す」の2つに分かれていた）
- 辻褄合わせ: 習慣型は経験値を、プロジェクト型は累計学習時間を、そのレベルの閾値ちょうどにする（表示レベルと実データが食い違い、次の学習で表示が戻るのを防ぐため）。**Lv1（5→1）は累計・経験値をともに0へ戻す（完全リセット）**
- プロジェクト型で目標未設定の場合は、累計では判定できないためレベルのみ動かす
- **本ボタンは開発用として、要件6.1「レベルは下がらない」を意図的に破る**（5→1へ戻す）。本番の成長処理はこの規則を守る
- **DBを実際に書き換える**（従来の表示プレビューと違い、確認後は自分で戻す必要がある）。演出をやり直したい場合は「今夜の学習時間を初期化」と併用する
- 将来 Lv1〜5 の灯り画像素材へ差し替える予定

## 5. 今夜の学習時間を初期化（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「今夜の学習時間を初期化」 |
| 挙動 | 当学習日の学習記録（`study_session`）と目標達成記録（`daily_goal_achievement`）を削除し、今夜の学習時間の合計を0へ戻す |
| 実装 | `src/app/index.tsx` の `DevPanel` / `src/db/repositories/devRepo.ts` |

**補足**

- 目的: **目標達成・休憩提案（要件5.1）はその学習日の実績合計に依存する**ため、確認をやり直すには合計を0へ戻す必要がある。本来のデータ初期化（要件10.10）は全データを消してしまい、確認のたびに初期設定からやり直すことになる
- **消すもの**: 今夜の学習記録と目標達成記録。学習記録に紐づくタグ（`session_tag`）はスキーマの `ON DELETE CASCADE` で連動削除される
- **消さないもの**:
  - **前夜以前の記録**（当学習日のぶんだけを消す）
  - **その夜の天気**（`daily_night_weather`）— 学習時間ではないため
  - **計測中のセッション**（`active_session`）— 稼働中のタイマーは止めない
  - **延長宣言のログ**（`additional_study_time`）— 学習時間ではなく、表示制御にも使われない（休憩提案の基準は `active_session` が持つ）
- **計測中に押した場合の注意**: 学習記録は消えるが、`active_session` の休憩提案の基準（`break_suggest_threshold_minutes`）は変わらない。「このまま続ける」等で基準を引き上げた後だと、初期化しても提案が出ないことがある。休憩提案を確認し直す場合は、初期化してから**新しくタイマーを開始する**（開始時に基準が一日の目標時間へ戻る）
- `devRepo.ts` は**リリース前にファイルごと削除する**（バレル `src/db/repositories/index.ts` の export も併せて外す）

## 6. レベルアップの閾値切り替え（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「レベルアップ: 5回/Lv(本番) ⇄ 1回/Lv(テスト)」 |
| 挙動 | 習慣型のレベルアップ閾値マスタ（`growth_level_threshold`）を、本番（Lv2=5/Lv3=10/Lv4=15/Lv5=20）とテスト（1/2/3/4）で切り替える |
| 実装 | `src/app/index.tsx` の `DevPanel` / `src/db/repositories/devRepo.ts` の `setHabitLevelStep()` |

**補足**

- 目的: 本番は**目標達成5回で1レベル**、Lv5（街完成）まで20日かかる（要件6.2①）。レベルアップ演出・完成演出を手で確認するのは現実的でないため、1回ごとに上がるよう一時的に下げる
- マスタを書き換えるのは、要件6.2が「基準値はマスタデータとして保持し、実装後のバランス調整で変更できるようにする」としているため。テスト用の抜け道を別に作るより本来の調整機構を使うほうが筋が通り、その機構が効くことの確認にもなる
- **注意: マスタはユーザーデータではないため、「データ初期化」では元に戻らない**（初期化は `DELETE FROM user` のみでマスタは残る）。本ボタンで本番へ戻すこと。新規インストールでは常にシード（本番）から始まる
- リリース前に本ボタンと `devRepo.ts` を撤去する。撤去時、マスタは各端末でシード値（本番）のはずだが、テスト中の端末が残っていないか一度 `getHabitLevelStep()` 相当で確認すること

## 7. 時刻の上書き・時刻を進める（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「時刻: 実時間 / 21:00 / 12:00」 |
| 挙動 | 押すたびに **実時間 → 21:00 → 12:00 → 実時間** と循環する。上書き中は時計の針・日時表示・**夜間帯判定（要件2.3）**・**学習日の集計**・**タイマーの計測と5:00判定**のすべてがその時刻で動く |
| 実装 | `src/lib/clock.ts`（時刻の実体）／ `src/app/index.tsx` の `DEV_CLOCK_HOURS` と `DevPanel`（切替UI） |

**「時刻を+30分進める」**

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「時刻を+30分進める」 |
| 挙動 | 押すたびにアプリ内の時刻を30分進める（`clock.ts` の `advanceDevTime`）。「時刻: 実時間」に戻すとオフセットは解除される |
| 目的 | **5:00自動終了**・**ポモドーロのフェーズ進行と全ループ完了**を、実際にその時刻まで待たずに確認するため |

**補足**

- 目的: 夜間帯（18:00〜翌5:00）の**内側と外側の両方**を、実際にその時刻まで待たずに確認するため
  - **21:00** = 夜間帯内 → 時計（学習開始ボタン）が**活性**
  - **12:00** = 夜間帯外 → 時計が**非活性**＋「この街が目覚めるのは 18:00 から」を表示
- **端末の時計は変更しない**（アプリが表示・判定に使う時刻だけを差し替える）
- 実体は `src/lib/clock.ts` の `nowMs()` に集約してある。アプリ内で現在時刻を使う箇所は**すべてここを経由する**ため、表示と判定が食い違わない
- 上書きは**実時間からのオフセット**として保持するため、合わせた後も時刻は自然に進む（21:00に合わせた5分後は21:05）
- **日付は実際の日付のまま**、時刻だけを合わせる。日付をまたぐ深夜帯（0:00〜5:00に前日へ帰属する動き）の確認には使えない
- 時刻を進めると計測中の実績学習時間も進む（時刻差分方式のため）。5:00を跨がせれば自動終了、ポモドーロの全ループ分を跨がせれば完了による自動終了を確認できる
- **オフセットはメモリ上にあり、アプリを再起動すると実時間へ戻る。** そのため**中断からの復元（要件3.2）の確認には使えない**（偽の時刻で開始したセッションを再起動すると、現在時刻が実時間へ戻り、経過が負→0に丸められて必ず「1分未満」で破棄されてしまう）。復元の確認は夜間帯に実時間で行うこと
