# 開発用テストボタン一覧

開発中の動作確認のために設置している UI と、暫定（仮実装）のまま残っている操作をまとめる。
**リリース前に「撤去チェックリスト」を必ず消化すること。**

`__DEV__` 限定のものは Expo の開発ビルドでのみ表示され、本番ビルドには出力されない。

---

開発用のボタンはホーム画面の下部に「開発用パネル」（`src/app/index.tsx` の `DevPanel`）としてまとめている。

---

## 1. データ初期化（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「開発用: データ初期化して初期設定へ」 |
| 挙動 | 全ユーザーデータを削除 → 設定を再読込 → 初期設定画面（`/setup`）へ遷移 |
| 実装 | `src/app/index.tsx` の `DevPanel` / `src/db/repositories/maintenanceRepo.ts` の `resetUserData()` |

**補足**

- 削除は `DELETE FROM user` のみ。スキーマの `ON DELETE CASCADE` により、ユーザーに紐づく全データ（学習記録・進捗・各種設定・マイタグ・計測状態等）が連動削除される。マスタデータは残る
- 初期設定フローを何度も試すために用意したもの
- 正式なデータ初期化（要件10.10・確認ダイアログ・タイマー稼働中の禁止を含む）は **Phase 6** で設定画面に実装する。その際に本ボタンは撤去してよい

---

## 2. Lv切り替え（開発用パネル）

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | `__DEV__` のみ |
| 場所 | ホーム画面 下部・開発用パネル内「Lv切り替え」 |
| 挙動 | **Lv5 プレビュー ⇄ 実際のレベル** をトグル。背景アートも連動して切り替わる（`lv5.png` ⇄ 実レベルの画像） |
| 実装 | `src/app/index.tsx` の `devLevelOverride` と `DevPanel` |

**補足**

- **DBは一切変更しない**（表示上のプレビューのみ）
- 目的: レベルに応じて育つ灯りの発光の確認と、**明るい Lv5 背景の上でも灯りが視認できるか**の確認
- 灯りの発光の調整値は `src/components/level-badge.tsx` 冒頭の定数（`CORE_OPACITY_*` / `HALO_*` / `SHADOW_*`）
- 将来 Lv1〜5 の灯り画像素材へ差し替える予定のため、その際に本プレビューの扱いを見直す

---

## 3. タイマーボタン（ホーム画面・右上のアナログ時計）※仮実装

| 項目 | 内容 |
| :---- | :---- |
| 表示条件 | ボタン自体は常時表示（時計は本来のUI）。**仮ダイアログは `__DEV__` のみ** |
| 場所 | ホーム画面 右上のアナログ時計 |
| 挙動 | 開発時: 仮のダイアログを表示する / 本番: 何も起きない |
| 実装 | `src/app/index.tsx` の `handleTimerPress` |

**補足**

- 仮ダイアログは `__DEV__` で囲っているため、本番ビルドでは表示されない（タップしても無反応）
- **Phase 3** でタイマー設定モーダル（S3）を開く処理に置き換える。あわせて夜間帯判定（要件2.3）もここで行う

---
